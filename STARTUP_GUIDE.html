ELITE SHINE ERP

System Startup & Shutdown Guide

Version 1.0 | January 2026

1. Prerequisites

Before starting the application, ensure you have the following installed:

  • Docker Desktop - For containerized deployment
  • Node.js 18+ - For local frontend development
  • Python 3.11+ - For local backend development
  • PostgreSQL 15 - Database (or use Docker)

2. Environment Setup

Step 1: Create Environment File

Copy the example environment file and configure it:

Command
copy .env.example .env

Step 2: Edit .env File

Configure these variables in your .env file:

Variable Description Example
POSTGRES_DB Database name eliteshine_erp
POSTGRES_USER Database username eliteshine_user
POSTGRES_PASSWORD Database password YourSecurePassword123!
SECRET_KEY Django secret key your-random-secret-key
VITE_API_BASE_URL API endpoint URL http://localhost:8000/api

3. Starting the Application

Option A: Using Docker (Recommended for Production)

This starts all services (Database, Backend, Frontend) together:

Start All Services
docker-compose up -d
View Logs
docker-compose logs -f
Note
First run may take several minutes to build images and download dependencies.
Service Port URL
Frontend 80 http://localhost
Backend API 8000 http://localhost:8000
Database 5432 localhost:5432

Option B: Manual Start (Development)

Step 1: Start Database

Using Docker for Database Only
docker-compose up -d db

Step 2: Start Backend

Navigate to Backend Directory
cd Bankend
Create Virtual Environment (First Time Only)
python -m venv venv
Activate Virtual Environment
venv\Scripts\activate
Install Dependencies (First Time Only)
pip install -r requirements.txt
Run Migrations
python manage.py migrate
Start Backend Server
python manage.py runserver 0.0.0.0:8000

Step 3: Start Frontend (New Terminal)

Navigate to Frontend Directory
cd frontend
Install Dependencies (First Time Only)
npm install
Start Development Server
npm run dev

Frontend will be available at: http://localhost:5173

4. Stopping the Application

Option A: Stop Docker Services

Stop All Services (Keep Data)
docker-compose down
Stop and Remove All Data
docker-compose down -v
Warning
Using -v flag will delete all database data permanently!

Option B: Stop Manual Services

Stop Backend
Press Ctrl + C in the backend terminal
Deactivate Virtual Environment
deactivate
Stop Frontend
Press Ctrl + C in the frontend terminal
Stop Database Container
docker-compose stop db

5. Quick Reference Commands

Action Command
Start all (Docker) docker-compose up -d
Stop all (Docker) docker-compose down
View logs docker-compose logs -f
Restart backend docker-compose restart backend
Rebuild containers docker-compose up -d --build
Check running containers docker ps
Backend shell docker exec -it eliteshine_backend bash
Database shell docker exec -it eliteshine_db psql -U eliteshine_user -d eliteshine_erp

6. Updating After Changes

Option A: Update with Docker (Production)

After making code changes, follow these steps to deploy updates:

Step 1: Pull Latest Changes (if using Git)
git pull origin main
Step 2: Rebuild and Restart Containers
docker-compose up -d --build
Quick Update Commands
  • Frontend only: docker-compose up -d --build frontend
  • Backend only: docker-compose up -d --build backend
  • Both without rebuild: docker-compose restart

Option B: Update Manual Development

Update Backend

Pull Changes
git pull origin main
Install New Dependencies (if requirements.txt changed)
cd Bankend && pip install -r requirements.txt
Run New Migrations (if models changed)
python manage.py migrate
Restart Backend Server
Press Ctrl + C then: python manage.py runserver 0.0.0.0:8000

Update Frontend

Pull Changes
git pull origin main
Install New Dependencies (if package.json changed)
cd frontend && npm install
Restart Dev Server
Press Ctrl + C then: npm run dev
Hot Reload
Frontend changes are automatically detected during development. No restart needed for most changes.

Git Workflow for Updates

Save Your Changes

Check What Changed
git status
Add All Changes
git add .
Commit Changes
git commit -m "Description of your changes"
Push to Remote
git push origin main

On Production Server

Pull and Deploy
git pull origin main && docker-compose up -d --build

7. Troubleshooting

Port Already in Use

netstat -ano | findstr :8000
taskkill /PID <PID> /F

Database Connection Failed

Ensure the database container is running:

docker-compose ps db

Frontend Not Loading

Clear npm cache and reinstall:

cd frontend && rm -rf node_modules && npm install

Reset Everything

docker-compose down -v && docker-compose up -d --build
